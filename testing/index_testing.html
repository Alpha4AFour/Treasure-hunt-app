Testing plan

Goals:

The primary goal of this plan is to implement a testing approach for evaluating the web application.
The plan outlines procedures for documenting the testing outcomes as well.

Description:

Treasure Hunt is an online game, where players take part in a quiz, using gps user data when playing the game.
Utilized features include geolocation, QR code scanner and provided API usage.

Testing purposes:

The main purpose of testing the app is to ensure that the functionality of Treashure Hunt game matches the specification brief.
The final outcome of the testing procudure should include:
Successfully tested application meeting the specifications and showing error-free performance.
Testing page, containing Unit Tests which will use Testing API to perform isolated testing of application specific features alongside user tests report.

Testers:

Testers duty is recording the result of testing on the testing page following the testing plan.
Testing team: [Timofei Svinarev, Boban Solomou].
Update testers list, if any changes occur.

Tests:

Functional test.

Purpose: Is to ensure that the output of the application matches the expected one, using Unit tests
Testers: Boban Solomou
Approach: implementation and usage of unit tests, which would use a Testing API to compare the expected and real outputs.

Exploratory

The type of tests used most of the time, as a result of the lack of time.
Purpose: The purpose of this type of test is to find and fix primary issues manually.
Approach: Exploratory testing is a type of investigating application without a predefined test plan,
allowing the tester to explore the application and identify bugs on the go.

User acceptance Test

Milestones: Getting games options, major functionality(receiving questions, sending answers), QR code scanner, Leaderboard, location
Tested OS: Windows 10, MacOS
Browser tests: Firefox, Chrome, Safari.

UAT entry criteria

All features must be implemented
All critical bugs must be fixed
Before UAT exploratory and functional testing should be done.

Test Cases:

Getting games options:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Expected result: You should have been redirected to the /app.html page with treasure hunt options
Getting login page:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Expected result: You should have been redirected to the /login.html page with name submission form
Starting game:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Enter your name in the form
  5. Expected result: If the name is already in use, alert will pop up with notification, otherwise the 1st question will be shown
Providing a correct answer:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Enter your name in the form, if it is taken provide a unique one
  5. Enter your answer.
  6. Expected result: In case the answer is correct, the score should update and notification "Well done!" should pop up, then next question should be shown.
Providing an incorrect answer:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Enter your name in the form, if it is taken provide a unique one
  5. Enter your answer.
  6. Expected result: In case the answer is incorrect, notification: "Wrong answer: [your answer]" should be shown and the score should update
Leaderboard:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Enter your name in the form, if it is taken provide a unique one
  5. Answer all questions to finish the game session.
  6. Expected result: You should be redirected to the leaderboard.html page where 10 players should be shown as a table.
QR code scanner:
  1. Visit: https://alpha4afour.github.io/Treasure-hunt-app/
  2. Start new game ( click on "Begin" button )
  3. Choose test option ( click on "Sample Treasure Hunt" button )
  4. Enter your name in the form, if it is taken provide a unique one
  5. Click "QR Code Scanner", provide access of camera to your browser (pop up window with camera expected)
  6. Scan a QR code with a camera
  7. Expected result: You should see a hint below the question.
